## 一、创建dataset.json

在nnUnet_raw/Dataset001_AISDmini中配置dataset.json文件：

```json
{
    "channel_names": {
        "0": "NCCT"
    },
    "labels": {
        "background": 0,
        "acute_infarct": 1
    },
    "numTraining": 3,
    "file_ending": ".nii.gz",
    "overwrite_image_reader_writer": "SimpleITKIO"
}
```

1、channel_names （模态定义）

- 单模态（只有CT）：`"0": "NCCT"` 或 `"CT"` 均可，名称只是标识
- 多模态（如CT+PET）：需 `"0": "CT"`, `"1": "PET"`，文件命名为 `xxx_0000.nii.gz` 和 `xxx_0001.nii.gz`
- **必须与文件名对应**：你的文件是 `_0000.nii.gz`，对应索引 `"0"`

2、labels（分割类别映射）

- **必须从0开始**：`0, 1, 2, 3...` 连续整数，不能跳过（如 `0, 2` 会报错）。
- 0必须是背景
- **类别名称**：可以是任意字符串（`"acute_infarct"`, `"lesion"`, `"tumor"`），只是描述。

AISD数据集是二分类任务，只有0和1。

3、numTraining（训练样本数）

必须等于实际imagesTr中的文件数。

4、file_ending（文件后缀）

标准格式必须带有==.gz==压缩后缀

5、overwrite_image_reader_writer(强制指定IO库)

"overwrite_image_reader_writer": "SimpleITKIO"

强制使用SimpleITK，确保读取的.nii.gz文件时格式正确。

## 二、数据预处理

nnUNetv2_plan_and_preprocess -d 001 --verify_dataset_integrity

nnUNetv2_plan_and_preprocess -d 001 --verify_dataset_integrity -np 1 #预处理阶段使用单进程处理

2D UNet与3D UNet对比：


| 维度            | 2D UNet                              | 3D UNet                             |
| --------------- | ------------------------------------ | ----------------------------------- |
| **输入数据**    | 单张或几张切片（512×512）           | 整个 3D Patch（如 128×128×64）    |
| **层间信息**    | ❌**丢失**（或仅局部通过多通道输入） | ✅**保留**（卷积核直接在 Z 轴滑动） |
| **显存需求**    | ✅**低**（4-6 GB 可跑）              | ❌**高**（需 8-12 GB+）             |
| **小病灶检测**  | ⚠️**可能漏检**（看不到上下文）     | ✅**更准确**（利用相邻层特征）      |
| **计算速度**    | ✅**快**（批量处理 2D 图像）         | ❌**慢**（3D 卷积计算量大）         |
| **AISD 适用性** | ⚠️**可用但次优**                   | ✅**推荐**（急性病灶通常跨多层）    |

因此，显卡大的话，尽量选择3D进行训练。

## 三、训练模型（至少5个数据）

1、nnUNetv2_train训练命令：

```python
nnUNetv2_train DATASET_ID CONFIGURATION FOLD [可选参数]
```

例如：nnUNetv2_train 001 3d_fullres 0 -tr nnUNetTrainer_5epochs

必选参数：

- `DATASET_ID`：数据集 ID（如 `001`）
- `CONFIGURATION`：配置名称（`2d`, `3d_fullres`, `3d_lowres`, `3d_cascade_fullres`）
- `FOLD`：交叉验证折数（`0`, `1`, `2`, `3`, `4` 或 `all`）

可选参数：

```python
-tr TRAINER_NAME          # 使用不同的训练器（默认 nnUNetTrainer）
-p PLANS_NAME             # 指定 plans 文件（默认 nnUNetPlans）
-pretrained_weights PATH  # 加载预训练权重
-num_gpus N              # 多 GPU 训练（默认 1）
-device DEVICE           # 指定设备（cuda/cpu/mps）
-val                     # 仅运行验证（不训练）
-c                       # 继续训练（从最后一个 checkpoint）
--npz                     # 保存 softmax 输出（用于集成）
--val_best               # 使用最佳模型而非最后模型验证
--disable_checkpointing  # 禁用保存 checkpoint（调试用）
```

2、Configuration详解

| 配置                     | 适用场景                                | 显存需求        | 特点                                 |
| ------------------------ | --------------------------------------- | --------------- | ------------------------------------ |
| **`2d`**                 | 切片较薄的 3D 数据（如你的 27 层 NCCT） | **低** (2-4GB)  | 将 3D 切分为 2D 切片训练，推理时堆叠 |
| **`3d_fullres`**         | 各向同性高分辨率 3D 数据                | **高** (8-16GB) | 原始分辨率，信息最完整               |
| **`3d_lowres`**          | 大体积数据（如全身 CT）                 | **中** (6-8GB)  | 先下采样训练，用于 cascade 第一阶段  |
| **`3d_cascade_fullres`** | 大体积 + 精细结构                       | **高**          | 结合 lowres 和 fullres，分阶段训练   |

3、Fold(交叉验证)

nnUNet 默认 **5-fold 交叉验证**：

- `0`, `1`, `2`, `3`, `4`：分别训练 5 个不同的训练/验证集划分
- `all`：用全部数据训练（不划分验证集，用于最终模型）

4、-tr 参数

除了默认的 `nnUNetTrainer`，常用变体：

**训练长度：**

- `nnUNetTrainer_5epochs`：仅训练 5 轮（快速验证代码/数据）
- `nnUNetTrainer_10epochs`：10 轮
- `nnUNetTrainer_20epochs`：20 轮
- `nnUNetTrainer_50epochs`：50 轮（默认是 1000 轮，早期停止通常在 200-600 轮）

**损失函数变体：**

- `nnUNetTrainerDiceLoss`：仅使用 Dice Loss（适合类别极不平衡）
- `nnUNetTrainerTopkLoss`：Top-k 损失（关注难样本）

**优化器变体：**

- `nnUNetTrainerAdam`：使用 Adam 替代默认的 SGD
- `nnUNetTrainerAdamW`：AdamW 优化器

**数据增强变体：**

- `nnUNetTrainerNoDeepSupervision`：禁用深度监督（简化网络）

5、验证模式（-val）

已训练完，只想在验证集上评估：

```python
nnUNetv2_train 001 2d 0 -val
# 或使用最佳模型
nnUNetv2_train 001 2d 0 -val --val_best
```

输出验证集 Dice 分数到控制台和日志文件。

## 四、一键训练模式
### 1、3D_fullres
（1）训练早停机制：
nnUNet 的早停机制（Early Stopping）在默认情况下，如果验证集 Dice 连续 50 轮没有提升，训练就会自动停止。
法一：可修改早停耐心值nnUNet\nnunetv2\training\nnUNetTrainer\nnUNetTrainer.py中的参数，再运行训练代码，fold0-fold4模型会接着训练（不用重新训练）
法二：新建文件 nnunetv2/training/nnUNetTrainer/variants/my_trainer.py：
```python
from nnunetv2.training.nnUNetTrainer.nnUNetTrainer import nnUNetTrainer

class nnUNetTrainerNoEarlyStop(nnUNetTrainer):
    def __init__(self, plans, configuration, fold, dataset_json, unpack_dataset, device):
        super().__init__(plans, configuration, fold, dataset_json, unpack_dataset, device)
        self.patience = 999  # 禁用早停
        self.num_epochs = 500  # 固定跑 500 轮
```
训练时运行：
```bash
nnUNetv2_train 001 3d_fullres 0 -tr nnUNetTrainerNoEarlyStop
```
（2）在anaconda终端直接运行以下代码，将五折交叉验证都训练完毕：
```bash
# ===== 环境设置 =====
$env:nnUNet_n_proc_DA="0"
$env:nnUNet_raw="C:\Users\Administrator\Desktop\nnUNet\nnUNet_raw"
$env:nnUNet_preprocessed="C:\Users\Administrator\Desktop\nnUNet\nnUNet_preprocessed"
$env:nnUNet_results="C:\Users\Administrator\Desktop\nnUNet\nnUNet_results"

# ===== 防止电脑睡眠 =====
powercfg /change monitor-timeout-ac 0
powercfg /change standby-timeout-ac 0
powercfg /change hibernate-timeout-ac 0

Write-Host "开始自动训练..." -ForegroundColor Green

# ===== 依次训练 5 个 fold =====
for ($fold=0; $fold -le 4; $fold++) {
    Write-Host "开始 Fold $fold / 4" -ForegroundColor Cyan
    nnUNetv2_train 001 3d_fullres $fold -tr nnUNetTrainer
    Write-Host "Fold $fold 完成！" -ForegroundColor Green
}

Write-Host "全部完成！" -ForegroundColor Green
```
### 2、2D训练
当数据集的z轴层数较小时，推荐采用2D进行训练。

```bash
# 1. 确认环境变量（如果窗口没关，应该还在）
$env:nnUNet_n_proc_DA="0"
$env:nnUNet_raw="C:\Users\Administrator\Desktop\nnUNet\nnUNet_raw"
$env:nnUNet_preprocessed="C:\Users\Administrator\Desktop\nnUNet\nnUNet_preprocessed"
$env:nnUNet_results="C:\Users\Administrator\Desktop\nnUNet\nnUNet_results"

# 2. 防止系统睡眠（过夜训练）
powercfg /change monitor-timeout-ac 0
powercfg /change standby-timeout-ac 0
powercfg /change hibernate-timeout-ac 0

Write-Host "开始 2D 训练（5-Fold）..." -ForegroundColor Green

# 3. 依次训练 5 个 fold（2D 配置）
for ($fold=0; $fold -le 4; $fold++) {
    Write-Host "`n========================================" -ForegroundColor Cyan
    Write-Host "开始 2D 训练 Fold $fold / 4" -ForegroundColor Cyan
    Write-Host "========================================" -ForegroundColor Cyan
    
    nnUNetv2_train 001 2d $fold -tr nnUNetTrainer
    
    Write-Host "`n✅ Fold $fold 完成！" -ForegroundColor Green
    Start-Sleep -Seconds 30  # 每 fold 间暂停 30 秒，释放 Windows DLL 缓存
}
```

## 五、训练中出现问题

1、卡在Epoch 0` / `Current learning rate: 0.01没有输出

出现问题：

- 任务管理器显示 **专用GPU内存占满**（3.8/4.0 GB）
- GPU **利用率为 0%**（显存占满但计算卡死）
- 按 `Ctrl+C` 后报错 `RuntimeError: CUDA out of memory` 或 `WinError 6`

解决方法：

修改nnunetv2/training/nnUNetTrainer/nnUNetTrainer.py文件：

==具体修改2d或3d_fullres的batch_size和patch_size==

```
"configurations":{
    "2d":{
        "batch_size": 1,
        "patch_size": [
        256,
        256
        ],
    }
    "3d_fullres":{
        "batch_size": 1,
        "patch_size": [
        256,
        256
        ],
    }

}
```
