## 二、检查image和mask是否匹配

检查维度


| 检查项                         | 风险                                                                     | 正确标准                                                   |
| ------------------------------ | ------------------------------------------------------------------------ | ---------------------------------------------------------- |
| **数据类型 (dtype)**           | CT 用 float32 会浪费空间且可能导致归一化问题；Mask 必须用 uint8 或 int16 | CT:`int16` 或 `float32` ， Mask:`uint8` (0,1,2...)         |
| **坐标方向 (Direction)**       | 方向矩阵不一致会导致左右/前后翻转，模型学到镜像特征                      | 必须与原始 DICOM 一致，通常是`[1,0,0, 0,1,0, 0,0,1]` (RAS) |
| **原点 (Origin)**              | 原点偏移会导致空间位置错位，影响滑动窗口推理                             | Image 和 Mask 的 Origin 必须完全相同                       |
| **强度范围 (Intensity Range)** | CT 值不在 HU 范围；Mask 值不是 0/1/2                                     | CT: -1024~+3000 HU，Mask: 0,1,2... (背景=0)                |
| **轴顺序 (Axis Order)**        | ZXY vs XYZ 混淆会导致层序颠倒或横竖翻转                                  | 必须是 (X, Y, Z) = ( sagittal, coronal, axial ) 或医学标准 |
| **Spacing 单位**               | 单位错误（如用米而不是毫米）会导致网络看到异常尺寸                       | 必须是 mm (毫米)，如`(0.5, 0.5, 5.0)`                      |
| **文件压缩**                   | 未压缩导致 raw/ 目录体积爆炸；过度压缩影响读取速度                       | 使用`.nii.gz` 格式（平衡体积和速度）                       |

脚本运行顺序：

check_aisd_alignment.py —> aisd_dicom2nifti.py -> aisd_png2nifti.py -> check_raw_alignment.py

生成文件：

spacing_info.json

validation_report.json

## 三、nnUNet目录下创建文件夹

nnUNet_raw

nnUNet_preprocess

nnUNet_results

### 1、nnUNet_raw（唯一需要手动准备的数据）

存放未经任何处理的原始医学图像和标注的数据集。

==nnUNet_raw/==
├── Dataset001_BrainTumor/          # 原始数据集1
│   ├── imagesTr/                   # 训练图像（原始CT/MRI）
│   │   ├── case_001_0000.nii.gz   # 原始CT图像（HU值未处理）
│   │   └── ...
│   ├── labelsTr/                   # 训练标签（原始mask）
│   │   ├── case_001.nii.gz        # 原始分割标注（0,1,2...）
│   │   └── ...
│   └── dataset.json                # 数据集配置文件（模态、类别数等）
└── Dataset002_Heart/               # 其他数据集...

==数据集文件夹命名格式：==

Dataset + 三位数ID + _ + 数据集名称
│         │           │    │
│         │           │    └─ 字母/数字/下划线（不能有空格、中文、连字符）
│         │           └──── 下划线分隔
└─────────┴──────────────── Dataset前缀 + 001-999的数字ID

特点：保留原始 HU 值（CT）或信号强度（MRI），保持原始的 spacing、方向、尺寸。

==单个数据文件命名格式：==

0091465_0000.nii.gz
│            │
│            └─ 通道索引 0000 → 对应 channel_names 中的 "0"
└─ 病例 ID

==多模态数据命名（共用一个标签）==：

输入:
├── Case_001_0000.nii.gz (CT)   ─┐
├── Case_001_0001.nii.gz (MRI)  ─┼─→ 网络 → 预测
└── Case_001_0002.nii.gz (PET)  ─┘

Label:
└── Case_001.nii.gz (0=背景, 1=肿瘤)  ← 三个模态共享同一个标注

### 2、nnUNet_preprocessed

存放重采样、归一化、裁剪后的标准化数据，供训练直接使用。

nnUNet_preprocessed/
├── Dataset001_BrainTumor/
│   ├── nnUNetPlans.json           # 生成的训练计划（网络结构配置）
│   ├── dataset_fingerprint.json   # 数据指纹（spacing、强度分布统计）
│   ├── dataset_properties.pkl      # 数据集属性缓存
│   └── nnUNetPlans_3d_fullres/    # 3d_fullres配置的具体数据
│       ├── case_001.npz           # 预处理后的压缩numpy数组（图像+标签）
│       ├── case_002.npz           # 已统一spacing、归一化、可能裁剪了非ROI区域
│       └── ...
└── ...

特点：由 nnUNetv2_plan_and_preprocess 命令自动生成，数据已统一格式，可直接输入网络训练。文件体积比raw小。

### 3、nnUnet_results

存放模型权重、训练日志、验证结果、推理结果。

nnUNet_results/
├── Dataset001_BrainTumor/
│   └── nnUNetTrainer__nnUNetPlans__3d_fullres/
│       ├── fold_0/                 # 第0折交叉验证结果
│       │   ├── checkpoint_final.pth    # 最终模型权重（训练完所有epoch）
│       │   ├── checkpoint_best.pth     # 最佳模型（验证集上Dice最高的模型）
│       │   ├── progress.png            # 训练曲线图（loss、Dice随epoch变化）
│       │   ├── training_log.txt      # 详细训练日志（每step的loss）
│       │   ├── validation_raw/       # 验证集预测结果（原始spacing还原后）
│       │   └── debug.json            # 调试信息
│       ├── fold_1/                 # 第1折（如进行5折交叉验证）
│       └── ...
└── ...

特点：由 nnUNetv2_train 自动生成，包含所有可复现实验的必要文件（模型+配置+日志），推理时（nnUNetv2_predict）自动来这里找模型权重。
